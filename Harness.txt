 Harness Engineering（线束工程）的完整能力体系，核心定位是以工程架构为核心驱动，构建可控、可观测、可复用的 AI 系统，而非单纯依赖提示词。

 一、核心定位
维度		Harness Engineering
核心驱动		以工程架构为核心
核心关注点		「如何构建系统」
核心优势		可控、稳定、可工程化

Harness Engineering 核心能力模块（图 1）
1、模型调度：多模型编排与路由策略，用声明式定义执行拓扑，实现多模型协同
2、执行环境：独立沙箱 + 上下文注入，完全可控的运行时状态，避免环境耦合
3、类型化约束：Schema 校验 + 结构化输出，编译期即可发现错误，解决黑盒输出问题
4、持久化管线：多步骤 DAG 工作流，支持状态持久化、断点续跑，突破单次调用限制
5、适配器架构：多环境统一抽象层，提供可移植、可替换组件，适配不同 API / 平台
6、全链路可观测：日志 / 追踪 / 指标三位一体，支持自动化测试与回归验证，解决黑盒不可控问题

二、全流程工程化落地
Harness Engineering 覆盖了从需求到生产的完整 DevOps 闭环，分为三大反馈环：
1. 内部反馈环（研发阶段）
需求定义：明确 Spec / 需求边界，为 AI 开发提供清晰输入
设计决策：通过 ADR（架构决策记录）做设计取舍，沉淀技术方案
编码实现：受 CLAUDE.md 规范约束，保证代码风格与质量统一
本地验证：自动化测试 / 回归 /smoke 测试，提前拦截问题
2. 推送反馈环（CI/CD 阶段）
变更门禁：62 项指标 + 26 项硬门槛，作为代码合并的前置校验
代码评审：规则策略 + 人工 review 双重把关
主干集成：支持 trunk merge
持续交付：14 条流程 / 29 个任务，自动化交付流水线，支持自动修复重试
3. 外部反馈环（生产阶段）
制品发布：4 条发布流程，标准化发版
预生产验证：预发验收 /smoke 测试
生产运行：真实流量 / 运行状态监控
监控演进：全链路监控 / 反馈闭环

三、Harness 核心架构与运行机制
Harness 作为 AI 智能体的「工程化底座」，由两大核心层构成，形成闭环控制：
1. Guides（引导层，前馈 feedforward）
为编码智能体提供前置约束与工具支持，从源头保证输出质量：
规则约束类：原则、约束 / 框架、规则、参考文档、操作指南
工具能力类：LSP 语言服务、CLI 与脚本、代码转换
2. Sensors（感知层，反馈 feedback）
为编码智能体提供运行时监控与纠错依据，实现自我迭代：
静态校验类：静态代码分析、评审智能体
运行观测类：日志、浏览器
3. 闭环逻辑
「引导层」通过前馈 (feedforward) 向编码智能体输出约束与工具，规范开发行为
「感知层」通过反馈 (feedback) 收集智能体运行数据，驱动自我纠错
底层算力支持：GPU 推理（AI 计算）+ CPU（工程化工具链）

四、核心价值总结
Harness Engineering 解决了纯 Prompt Engineering 的核心痛点：
✅ 从黑盒到白盒：全链路可观测、可校验、可测试，告别人工排查
✅ 从单次到持久：支持复杂工作流、断点续跑，适配企业级长流程任务
✅ 从耦合到解耦：适配器架构实现多模型 / 多环境兼容，系统可扩展、可维护
✅ 从零散到工程化：融入完整 DevOps 流程，让 AI 开发符合企业级软件工程标准
