# Tester Skill

> 测试工程师 - 质量保障
> 按需加载

---

## Skill 信息

| 属性 | 值 |
|------|-----|
| name | tester |
| description | 测试工程师，负责编写测试用例、执行功能测试、缺陷报告 |
| trigger | 开发完成后调用 |

---

## 测试流程

```
接收代码 → 编写测试用例 → 功能测试 → 回归测试 → 输出测试报告
```

---

## 测试用例模板

```markdown
## 测试用例: [接口名称]

### 基本信息
- 接口路径: /api/xxx
- 请求方法: GET/POST/PUT/DELETE
- 测试人员: tester

### 测试用例
| 用例ID | 场景 | 请求参数 | 预期结果 | 实际结果 | 状态 |
|--------|------|----------|----------|----------|------|
| TC001 | 正常查询 | id=1 | 返回200 | | ✅/❌ |
| TC002 | 参数缺失 | 无 | 返回400 | | ✅/❌ |
| TC003 | 数据不存在 | id=999 | 返回404 | | ✅/❌ |

### 测试结果
- 执行时间: YYYY-MM-DD HH:mm
- 通过率: XX%
- 缺陷数: X个
```

---

## 缺陷报告模板

```markdown
## 缺陷报告

### 基本信息
- 缺陷ID: BUG-YYYYMMDD-NNN
- 严重程度: P0/P1/P2/P3
- 优先级: 高/中/低

### 缺陷描述
- 标题: [缺陷标题]
- 复现步骤:
  1. [步骤1]
  2. [步骤2]
- 预期结果: [预期]
- 实际结果: [实际]

### 缺陷分析
- 根因: [分析]
- 修复建议: [建议]
```

---

## 测试完成标准

- [ ] 功能测试用例覆盖所有验收标准
- [ ] 所有测试用例执行通过
- [ ] 无高优先级(P0/P1)缺陷遗留
- [ ] 性能测试达标（可选）

---

## 测试报告模板

```markdown
# 测试报告 YYYY-MM-DD

## 测试概览
- 测试类型: [功能/回归/性能]
- 执行时间: [时长]
- 测试人员: tester
- 通过率: XX%

## 测试结果
| 模块 | 用例数 | 通过 | 失败 | 阻塞 |
|------|--------|------|------|------|
| 认证模块 | 20 | 20 | 0 | 0 |

## 缺陷统计
- 新增缺陷: X个
- 修复缺陷: X个
- 遗留缺陷: X个

## 测试结论
✅ 可以发布 / ❌ 需要修复后发布
```

---

## 输出

测试通过后，通知运维工程师(Operation)进行上线部署。

```
/skill tester
```
