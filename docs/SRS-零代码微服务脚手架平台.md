# RuoYi-Cloud-Nocode 零代码微服务平台

> 后端项目名：RuoYi-Cloud-Nocode
> 前端项目名：Ui-Nocode

## 一、需求背景与目标

### 1.1 业务背景

当前企业级软件开发面临以下痛点：
- **开发效率低**：传统开发模式需要编写大量重复性代码（CRUD、实体类、Service、Controller等）
- **技术门槛高**：微服务架构复杂度高，分布式事务、服务治理、配置管理等需要专业知识
- **维护成本大**：多模块项目代码冗余，升级困难
- **扩展性差**：业务模块耦合度高，难以快速适配不同行业场景

### 1.2 核心目标

**主目标**：打造一款基于Spring生态的**零代码微服务平台（RuoYi-Cloud-Nocode）**，采用 **Skill + LLM + Json2UI + Agent + 代码知识库** 核心技术架构，通过智能体和知识库实现复杂微服务项目的自动化构建：

1. **零代码快速构建**：通过配置化+AI智能生成完整微服务模块
2. **热插拔热启动**：采用Spring插件技术，实现业务模块的动态加载与卸载
3. **一站式企业级解决方案**：集成认证授权、工作流、报表、支付等常用企业能力
4. **多行业适配**：通过模块化设计支持不同业务场景的灵活组合
5. **AI智能开发**：LLM理解需求 → 匹配代码模板 → Skill调度 → 自动生成代码

### 1.3 范围界定

| 包含内容 | 不包含内容 |
|---------|-----------|
| 微服务脚手架核心框架 | 行业专用业务模块（如医疗HIS、电商OMS等） |
| 插件机制与热插拔能力 | 第三方支付接口对接 |
| 零代码配置平台前端 | 定制化AI能力 |
| 代码生成引擎 | 移动端原生应用 |
| 基础企业功能（用户、角色、部门、字典、配置、日志等） | SaaS多租户完整实现 |
| 工作流引擎集成 | 大数据/AI分析能力 |

### 3.2.1 技术底座

#### 3.2.1.1 Liquor 动态编译引擎

Liquor是一个高性能的Java动态编译与执行框架，为平台提供运行时动态代码生成和执行能力。

| 特性 | 说明 |
|------|------|
| **即时编译(JIT)** | 支持Java源码即时编译为Class文件，无需重启应用 |
| **热加载** | 代码修改后可立即生效，支持类的热替换 |
| **沙箱执行** | 提供安全的类加载隔离，防止恶意代码执行 |
| **缓存优化** | 编译结果缓存，避免重复编译开销 |
| **依赖管理** | 智能处理运行时依赖，支持Maven坐标自动解析 |

| 能力 | 说明 | 应用场景 |
|------|------|---------|
| **动态类生成** | 运行时创建Java类 | AI生成代码的动态加载 |
| **方法执行** | 动态执行编译后的方法 | 动态业务逻辑执行 |
| **脚本执行** | 支持Groovy/JavaScript混合执行 | 轻量级脚本处理 |
| **表达式计算** | 支持复杂数学/逻辑表达式 | 数据加工、计算字段 |

#### 3.2.1.2 PF4J + MyBatis + Liquor 融合架构

Spring Boot + PF4J + MyBatis 与 Liquor 无冲突，反而能互补。PF4J 做「静态模块拆分」，Liquor 做「模块内逻辑动态更新」，MyBatis 做「统一数据访问」，三者形成完整的动静结合架构。

| 层次 | 技术 | 职责 | 特点 |
|------|------|------|------|
| **模块化** | **PF4J** | 静态模块拆分（插件化加载） | 物理分离、独立部署、类加载器隔离 |
| **动态逻辑** | **Liquor** | 模块内业务逻辑热更新 | 运行时编译、即时生效、无需重启 |
| **数据访问** | **MyBatis** | 统一数据访问层 | 稳定持久层、SQL管控、映射能力 |

**架构互补原理**：

```
┌─────────────────────────────────────────────────────────────────┐
│                    插件模块 (PF4J)                              │
├─────────────────────────────────────────────────────────────────┤
│  ┌─────────────────┐    ┌─────────────────┐                   │
│  │  静态Java代码   │    │  动态业务逻辑   │                   │
│  │  (Controller/   │    │  (Liquor加载   │                   │
│  │   Service)      │    │   的规则/Script)│                   │
│  └────────┬────────┘    └────────┬────────┘                   │
│           │                     │                             │
│           └──────────┬──────────┘                             │
│                      ▼                                         │
│           ┌─────────────────────┐                            │
│           │   MyBatis数据访问    │                            │
│           │   (统一持久层)       │                            │
│           └─────────────────────┘                            │
└─────────────────────────────────────────────────────────────────┘
```

| 组合场景 | 说明 |
|---------|------|
| PF4J + Liquor | PF4J 加载插件框架，Liquor 实现插件内业务逻辑热更新 |
| PF4J + MyBatis | PF4J 管理插件模块，MyBatis 提供统一的数据访问 |
| PF4J + Liquor + MyBatis | 完整架构：插件化加载 + 逻辑热更新 + 数据持久化 |

| 能力 | 说明 | 应用场景 |
|------|------|---------|
| **插件内热更新** | 插件内部业务逻辑通过 Liquor 动态更新 | 审批规则、计算公式、校验逻辑 |
| **插件隔离** | PF4J 类加载器隔离 + Liquor 沙箱执行 | 多租户隔离、安全防护 |
| **动态数据源** | Liquor 动态执行 SQL + MyBatis 映射 | 动态报表、自定义查询 |

#### 3.2.1.3 QLExpress 规则引擎

QLExpress是阿里巴巴开源的规则引擎，为平台提供可视化、可配置的规则定义和执行能力。

| 特性 | 说明 |
|------|------|
| **表达式语法** | 类似Java语法，上手简单 |
| **高性能** | 编译后执行，性能接近原生Java |
| **可视化配置** | 支持规则可视化配置和管理 |
| **热更新** | 规则修改无需重启，即时生效 |
| **调试支持** | 提供规则执行日志和断点调试 |

| 能力 | 说明 | 应用场景 |
|------|------|---------|
| **条件规则** | IF-THEN条件判断 | 审批流程、业务校验 |
| **计算规则** | 四则运算、函数调用 | 费用计算、数据加工 |
| **集合操作** | 遍历、过滤、映射 | 批量数据处理 |
| **外部函数** | 注册Java方法为规则函数 | 扩展规则能力 |

```java
// QLExpress规则示例
// 业务场景：订单金额计算规则
String rule = "
    // 基础金额
    baseAmount = order.amount;
    // 会员折扣
    discount = 1.0;
    if (user.level == 'VIP') {
        discount = 0.8;
    } else if (user.level == 'Gold') {
        discount = 0.9;
    }
    // 优惠券
    couponDiscount = 0;
    if (order.couponCode != null) {
        couponDiscount = getCouponDiscount(order.couponCode);
    }
    // 最终金额
    finalAmount = baseAmount * discount - couponDiscount;
    return finalAmount;
";
```

## 3.7 代码知识库系统（Code RAG）

### 3.7.1 设计理念

传统LLM代码生成面临的挑战：
- **Token消耗大**：每次都重新生成完整代码
- **准确率问题**：LLM可能生成不规范的代码
- **风格不一致**：与现有代码风格不统一
- **重复造轮子**：相同功能的代码重复开发

**解决方案**：构建代码知识库 + RAG检索增强生成

```
用户需求 -> 向量化检索 -> 匹配代码模板 -> LLM优化 -> 最终代码
                        ↓
              代码知识库（Code Template Store）
```

### 3.7.2 知识库架构

```
┌─────────────────────────────────────────────────────────────┐
│                    代码知识库系统                             │
├─────────────────────────────────────────────────────────────┤
│  ┌─────────────┐  ┌─────────────┐  ┌─────────────┐        │
│  │代码采集层   │  │向量化存储层 │  │检索匹配层   │        │
│  │             │  │             │  │             │        │
│  │•模板仓库    │  │•向量数据库  │  │•语义检索    │        │
│  │•优秀案例    │  │•ES全文索引  │  │•关键词匹配  │        │
│  │•代码片段    │  │•图数据库    │  │•混合检索    │        │
│  │•SDK代码     │  │•PostgreSQL  │  │•相似度计算  │        │
│  └─────────────┘  └─────────────┘  └─────────────┘        │
│          │                │                │                │

### 3.7.3 知识库类型

| 类型 | 说明 | 示例 |
|------|------|------|
| **模板库** | 标准化代码模板 | CRUD模板、Controller模板、Service模板 |
| **组件库** | 可复用组件代码 | 文件上传组件、字典组件、表格组件 |

  step2: "向量化检索 - 在知识库中搜索相似模板"
  step3: "模板选择 - 选择最匹配的代码模板"
  step4: "参数注入 - 将需求参数注入模板"
  step5: "LLM优化 - LLM优化生成最终代码"
  step6: "质量校验 - 校验代码语法和风格"
```

### 3.7.6 RAG检索流程

```
1. 用户输入需求
   ↓
2. 需求解析
   - 提取实体：Customer（客户）
   - 提取动作：CRUD（增删改查）
   - 提取上下文：Spring Boot + MyBatis-Plus
   ↓
3. 向量化处理
   - 将需求转换为向量
   - 使用text-embedding-ada-002或本地向量化模型
   ↓
4. 知识库检索
   - 向量相似度搜索（Milvus/Pgvector）
   - 关键词过滤
   - 重排序（Rerank）
   ↓
5. 模板匹配结果
   - 匹配度95%：crud_controller_template
   - 匹配度87%：rest_api_template
   - 匹配度65%：generic_controller
   ↓
6. 代码生成
   - 选用95%匹配模板
   - 注入参数：entityName=Customer
   - LLM优化注释和异常处理
   ↓
7. 最终代码输出
```

### 7.5 核心技术实现要点

| 模块 | 实现要点 |
|------|---------|
| **ruoyi-nocode-model** | 动态Entity映射表结构，运行时加载 |
| **ruoyi-nocode-gen** | Velocity模板 + AST分析，支持热更新 |
| **ruoyi-nocode-ai** | Skill调度 + ReAct Agent + 向量检索 |
| **ruoyi-nocode-api** | 动态SQL构建 + 多数据源路由 |
| **ruoyi-nocode-plugin** | 自定义类加载器，实现插件隔离 |
| **Json2UI** | Vue3动态组件 + VM2沙箱执行事件 |
