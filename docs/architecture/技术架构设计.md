## 三、核心模块架构

### 3.1 PF4J + Liquor + MyBatis 融合架构

这是本平台的核心技术底座，实现「静态模块拆分」+「动态逻辑更新」+「统一数据访问」的融合。

#### 3.1.1 架构分层

```
┌─────────────────────────────────────────────────────────────────┐
│                    插件模块 (PF4J)                              │
├─────────────────────────────────────────────────────────────────┤
│  ┌─────────────────┐    ┌─────────────────┐                   │
│  │  静态Java代码   │    │  动态业务逻辑   │                   │
│  │  (Controller/   │    │  (Liquor加载   │                   │
│  │   Service)      │    │   的规则/Script)│                   │
│  └────────┬────────┘    └────────┬────────┘                   │
│           │                     │                             │
│           └──────────┬──────────┘                             │
│                      ▼                                         │
│           ┌─────────────────────┐                            │
│           │   MyBatis数据访问    │                            │
│           │   (统一持久层)       │                            │
│           └─────────────────────┘                            │
└─────────────────────────────────────────────────────────────────┘
```

#### 3.1.2 技术职责

| 技术 | 职责 | 特点 |
|------|------|------|
| **PF4J** | 静态模块拆分 | 物理分离、独立部署、类加载器隔离 |
| **Liquor** | 模块内逻辑热更新 | 运行时编译、即时生效、无需重启 |
| **MyBatis** | 统一数据访问 | 稳定持久层、SQL管控、映射能力 |

#### 3.1.3 组合场景

| 组合 | 场景 |
|------|------|
| PF4J + Liquor | PF4J 加载插件框架，Liquor 实现插件内业务逻辑热更新 |
| PF4J + MyBatis | PF4J 管理插件模块，MyBatis 提供统一的数据访问 |
| PF4J + Liquor + MyBatis | 完整架构：插件化加载 + 逻辑热更新 + 数据持久化 |
