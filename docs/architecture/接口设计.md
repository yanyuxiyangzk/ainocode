## 一、接口规范

### 1.1 RESTful风格

| 方法 | 说明 | 示例 |
|------|------|------|
| GET | 查询 | GET /user/{id} |
| POST | 新增 | POST /user |
| PUT | 更新 | PUT /user/{id} |
| DELETE | 删除 | DELETE /user/{id} |

### 1.2 统一响应格式

```json
{
  "code": 200,
  "msg": "操作成功",
  "data": {
    // 业务数据
  },
  "rows": [],
  "total": 0
}
```

| 字段 | 类型 | 说明 |
|------|------|------|
| code | int | 状态码 |
| msg | String | 消息 |
| data | Object | 响应数据 |
| rows | Array | 列表数据 |
| total | int | 总数 |

### 1.3 状态码定义

| 状态码 | 说明 |
|--------|------|
| 200 | 操作成功 |
| 201 | 创建成功 |
| 204 | 删除成功 |
| 400 | 请求参数错误 |
| 401 | 未授权 |
| 403 | 无权限 |
| 404 | 资源不存在 |
| 409 | 资源冲突 |
| 500 | 服务器内部错误 |

## 二、核心接口

### 2.1 认证接口

#### 2.1.1 登录

```
POST /auth/login
```

**请求参数：**
```json
{
  "username": "admin",
  "password": "admin123"
}
```

**响应：**
```json
{
  "code": 200,
  "msg": "操作成功",
  "data": {
    "token": "eyJhbGciOiJIUzI1NiJ9...",
    "expire": 3600
  }
}
```

#### 2.1.2 登出

```
POST /auth/logout
```

### 2.2 用户管理接口

#### 2.2.1 用户列表

```
GET /system/user/list
```

**查询参数：**
| 参数 | 类型 | 说明 |
|------|------|------|
| userName | String | 用户名（模糊） |
| phonenumber | String | 手机号 |
| status | String | 状态 |
| deptId | Long | 部门ID |
| pageNum | Int | 页码 |
| pageSize | Int | 每页条数 |

#### 2.2.2 用户详情

```
GET /system/user/{userId}
```

#### 2.2.3 新增用户

```
POST /system/user
```

**请求参数：**
```json
{
  "userName": "test",
  "nickName": "测试用户",
  "deptId": 100,
  "phonenumber": "13800138000",
  "sex": "0",
  "status": "0",
  "roleIds": [1, 2]
}
```

#### 2.2.4 修改用户

```
PUT /system/user
```

#### 2.2.5 删除用户

```
DELETE /system/user/{userIds}
```

## 三、接口安全

### 3.1 认证方式

- 基于Token的认证（Sa-Token）
- Token放在Header中：`Authorization: Bearer <token>`

### 3.2 权限控制

| 方式 | 说明 |
|------|------|
| 接口级 | 通过@PreAuthorize控制 |
| 按钮级 | 前端按钮绑定权限标识 |
| 数据级 | 行级数据权限控制 |

### 3.3 接口限流

| 策略 | 配置 |
|------|------|
| 全局限流 | 网关层统一限流 |
| IP限流 | 针对IP的限流 |
| 用户限流 | 针对登录用户的限流 |

## 四、版本控制

### 4.1 接口版本

```
/api/v1/user/list
/api/v2/user/list
```

### 4.2 版本兼容

| 策略 | 说明 |
|------|------|
| 向下兼容 | 新增字段不影响旧版本 |
| 废弃标记 | 废弃接口添加@Deprecated |
| 过渡期 | 至少保留2个版本 |
