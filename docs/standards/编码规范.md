## 一、命名规范

### 1.1 包命名

| 类型 | 规范 | 示例 |
|------|------|------|
| 包名 | 小写字母 | com.ruoyi.nocode |
| 模块包 | 小写+模块名 | com.ruoyi.nocode.plugin |
| 业务包 | 小写+业务名 | com.ruoyi.nocode.customer |

### 1.2 类命名

| 类型 | 规范 | 示例 |
|------|------|------|
| Controller | Controller结尾 | UserController |
| Service | Service结尾 | UserService |
| ServiceImpl | ServiceImpl结尾 | UserServiceImpl |
| Mapper | Mapper结尾 | UserMapper |
| Entity | 实体名 | User |
| DTO | DTO结尾 | UserDTO |
| VO | VO结尾 | UserVO |
| Enum | Enum结尾 | UserStatusEnum |

### 1.3 方法命名

| 类型 | 规范 | 示例 |
|------|------|------|
| 查询 | get/find/select开头 | getUserById |
| 新增 | save/insert/add开头 | saveUser |
| 更新 | update/modify开头 | updateUser |
| 删除 | delete/remove开头 | deleteUser |
| 条件查询 | findBy开头 | findByName |
| 列表 | list开头 | listUser |
| 统计 | count/sum开头 | countUser |

### 1.4 变量命名

| 类型 | 规范 | 示例 |
|------|------|------|
| 普通变量 | 小驼峰 | userName, userList |
| 常量 | 全大写下划线 | MAX_COUNT, DEFAULT_STATUS |
| 成员变量 | 小驼峰 | （同普通变量，加this.区分） |
| 静态变量 | s开头大驼峰 | sInstance |

## 二、代码格式

### 2.1 缩进与空格

- 使用4个空格缩进（不使用Tab）
- 运算符前后各1个空格
- 逗号后1个空格
- 大括号前不换行

```java
// 正确示例
if (user != null && user.getStatus() == 1) {
    doSomething();
}

// 错误示例
if(user!=null&&user.getStatus()==1){
    doSomething();
}
```

### 2.2 行长度

- 单行代码不超过120个字符
- 过长内容换行缩进

```java
// 正确示例
String url = "http://example.com/api/v1/users"
        + "/list?page=1&size=10";
```

### 2.3 括号使用

- 左大括号不换行
- 右大括号单独一行
- 空方法体用大括号

```java
// 正确
public void doNothing() { }

// 错误
public void doNothing()
{ }
```

## 四、异常处理

### 4.1 异常捕获

- 不捕获Exception/Throwable（除非必要）
- 优先捕获具体异常
- 异常要记录日志

```java
// 正确示例
try {
    userService.save(user);
} catch (DuplicateKeyException e) {
    log.error("用户已存在: {}", user.getName());
    throw new BusinessException("用户已存在");
} catch (Exception e) {
    log.error("保存用户失败", e);
    throw new BusinessException("保存失败");
}
```

### 4.2 异常抛出

- 使用自定义业务异常
- 避免暴露内部异常信息

## 五、数据库操作

### 5.1 SQL规范

- 关键字大写
- 表名、字段名使用小写
- 使用参数化查询

```sql
-- 正确
SELECT user_id, user_name FROM sys_user WHERE status = '0';

-- 错误
select user_id,user_name from sys_user where status='0';
```

### 5.2 MyBatis规范

- 使用#{}占位符，不使用${}（除排序字段）
- 明确指定返回类型
- 添加合理的注释

```xml
<!-- 正确 -->
<select id="selectUser" resultType="User">
    SELECT * FROM sys_user WHERE user_id = #{userId}
</select>

<!-- 错误 -->
<select id="selectUser" resultType="User">
    SELECT * FROM sys_user WHERE user_id = ${userId}
</select>
```

## 六、日志规范

### 6.1 日志级别

| 级别 | 使用场景 |
|------|---------|
| ERROR | 系统异常、影响业务 |
| WARN | 潜在问题、可恢复异常 |
| INFO | 重要业务节点 |
| DEBUG | 开发调试信息 |

### 6.2 日志格式

```java
// 正确
log.info("用户登录成功: userId={}, username={}", userId, username);
log.error("保存用户失败", e);

// 错误
log.info("用户登录成功");
log.error(e.getMessage());
```

## 八、事务规范

### 8.1 事务传播

| 传播行为 | 说明 |
|---------|------|
| REQUIRED | 有事务则加入，无则创建 |
| REQUIRES_NEW | 总是创建新事务 |
| NOT_SUPPORTED | 不使用事务 |

### 8.2 事务使用

```java
@Transactional(rollbackFor = Exception.class)
public void saveUser(User user) {
    // 业务逻辑
}
```

## 九、接口规范

### 9.1 返回值

```java
// 正确
public AjaxResult save(User user) {
    // 成功
    return AjaxResult.success();
    // 失败
    return AjaxResult.error("保存失败");
}

// 列表
return AjaxResult.success(list, total);
```

### 9.2 参数校验

```java
// 使用校验注解
@NotNull(message = "用户ID不能为空")
private Long userId;

@NotBlank(message = "用户名不能为空")
@Size(min = 2, max = 20)
private String userName;
```
