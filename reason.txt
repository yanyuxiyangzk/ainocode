# 24小时自动开发Agent团队设计方案 v2.0

> 目标：构建一个自主运行的AI开发团队，24小时不间断推进项目开发
> 项目背景：RuoYi-Cloud-Nocode 零代码微服务平台（PF4J+Liquor动态编译）
> 设计日期：2026-04-02
> 版本：v2.0（增强版）

---

## 一、核心设计理念

### 1.1 为什么需要自动化开发？

| 传统开发痛点 | AI自动化优势 |
|-------------|-------------|
| 人类需要睡眠 | 24小时不间断运行 |
| 会疲劳、遗忘 | 不会疲劳、记忆持久化 |
| 上下文切换慢 | 并行处理多任务 |
| 情绪影响效率 | 稳定输出、可量化 |
| 沟通成本高 | 自主决策、无需协调 |

### 1.2 你期望的核心能力

```
┌────────────────────────────────────────────────────────────┐
│                    你描述的场景                              │
├────────────────────────────────────────────────────────────┤
│  1. 24小时运行，自己判断下一步做什么                           │
│  2. 不需要你下指令，自主规划和执行                            │
│  3. 有规则约束，不乱来                                      │
│  4. 遇到关键决策能主动问你                                   │
│  5. 基于Claude Code工具实现                                 │
└────────────────────────────────────────────────────────────┘
```

### 1.3 设计的核心挑战与解决方案

| 挑战 | 解决方案 |
|------|----------|
| 如何判断"下一步做什么" | 任务优先级算法 + 项目状态分析 |
| 如何防止走偏 | 规则约束 + 质量门卫 + 用户监督点 |
| 如何持久化记忆 | MEMORY.md + 任务队列文件 |
| 如何24小时运行 | Cron定时触发 + 循环机制 |

### 2.2 Agent详细职责

#### 2.2.1 Super Orchestrator（主控）

```markdown
角色定位：项目CEO + 智能决策者

核心职责：
1. 【感知】读取项目状态
   - 读取 PROJECT_MEMORY.md 了解项目背景
   - 读取 PROJECT_RULES.md 了解技术要求
   - 读取 tasks/backlog.md 了解待办任务
   - 分析 git log 了解最近变更

2. 【决策】判断下一步行动
   - 任务优先级算法（紧急度 × 复杂度 × 依赖关系）
   - 决定是继续任务、规划新任务、还是提问
   - 风险评估：高风险操作必须停等用户

3. 【执行】调度专业Agent
   - 分配任务给 Backend/Frontend/DevOps Agent
   - 监控执行进度和质量
   - 处理异常和阻塞

4. 【闭环】质量门卫
   - 编译必须通过：mvn compile
   - 测试必须通过：mvn test
   - 不通过则回退或重新尝试

5. 【报告】定期汇报
   - 每小时简短进度
   - 每天开发日报
   - 每周里程碑总结
```

### 3.2 智能决策引擎

```python
# Orchestrator 决策流程伪代码

def decide_next_action():
    """
    决策算法：判断下一步应该做什么
    """

    # 步骤1: 检查是否有阻塞任务需要用户决策
    blocked = load_blocked_tasks()
    if blocked:
        for task in blocked:
            if task.needs_user_decision:
                return ASK_USER(task)  # 需要用户决策

    # 步骤2: 检查是否有正在进行的任务
    in_progress = load_in_progress_tasks()
    if in_progress:
        for task in in_progress:
            if task.is_stuck:
                return ANALYZE_BLOCKER(task)  # 分析阻塞原因
            elif task.is_done:
                return QUALITY_GATE(task)  # 质量检查

    # 步骤3: 从待办池取出最高优先级任务
    backlog = load_backlog_tasks()
    if backlog:
        task = select_by_priority(backlog)
        return ASSIGN_TASK(task)

    # 步骤4: 没有任务了，规划新任务
    return PLAN_NEW_TASKS()


def priority_algorithm(task):
    """
    优先级计算公式：
    Priority = (Urgency × 0.4) + (Impact × 0.3) + (Readiness × 0.3)

    - Urgency: 紧急度 (1-10)
      - P0核心功能 = 10
      - P1重要功能 = 7
      - P2普通功能 = 4

    - Impact: 影响范围 (1-10)
      - 核心模块 = 10
      - 公共模块 = 7
      - 辅助模块 = 4

    - Readiness: 就绪度 (1-10)
      - 依赖已就绪 = 10
      - 部分就绪 = 6
      - 阻塞 = 2
    """
    urgency = task.urgency_score
    impact = task.impact_score
    readiness = task.dependency_ready_score

    return (urgency * 0.4) + (impact * 0.3) + (readiness * 0.3)
```

### 3.4 用户交互机制

#### 必须停等用户的情况

```markdown
🚨 【硬性停止条件】 - 必须等待用户回复才能继续

1. 🔴 删除/修改核心架构
   - 删除整个模块
   - 修改PF4J/Liquor核心逻辑
   - 修改数据库Schema

2. 🔴 不可逆的重大决策
   - 更换核心技术栈
   - 删除已开发功能
   - 修改API兼容性

3. 🔴 连续失败3次的任务
   - 同一任务反复失败
   - 需要人工介入排查

4. 🟡 不明确的业务逻辑
   - 需求描述模糊
   - 可能有歧义的实现方式

5. 🟡 Token消耗异常
   - 单次循环消耗 > 200k tokens
   - 累计日消耗 > 1M tokens
```

### 3.5 循环运行机制

```python
# 自动循环调度（基于Cron定时触发）

CRON_SCHEDULE = """
# 每10分钟检查一次项目状态
*/10 * * * *  claude-code --agent orchestrator --mode check

# 每小时执行一次任务（如果有）
0 * * * *    claude-code --agent orchestrator --mode execute

# 每天早上9点生成日报（工作日）
0 9 * * 1-5  claude-code --agent orchestrator --mode daily_report

# 每周一早上9点生成周报
0 9 * * 1    claude-code --agent orchestrator --mode weekly_report
"""

# Orchestrator 主循环
def orchestrator_loop(mode="check"):
    if mode == "check":
        # 轻量级检查：任务状态、是否有阻塞
        check_task_health()
        check_for_blockers()
        decide_if_needs_execution()

    elif mode == "execute":
        # 完整执行：决策 + 分配 + 质量检查
        execute_task_cycle()

    elif mode == "daily_report":
        # 生成日报
        generate_daily_summary()

    elif mode == "weekly_report":
        # 生成周报
        generate_weekly_summary()
```
